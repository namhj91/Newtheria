# 08. 카드 선택 연출 규칙


## 1) 개요
- 게임 중 선택지는 카드(Card) 형태로 표현한다.
- 카드는 기본적으로 팬(Fan) 형태의 부채꼴 레이아웃으로 펼쳐진다.

## 2) 카드 규격(비율)
- 기본 카드는 **가로:세로 = 1:1.618(황금비)**의 세로형 비율을 사용한다.
- 시작 화면 카드 선택지와 이벤트 카드 선택지는 동일한 카드 베이스를 공유한다.

## 3) 기본 선택 흐름
1. 선택지 후보 생성
2. 카드가 부채꼴로 펼쳐짐
3. 유저가 카드 Hover/Focus
4. 카드 1장 선택 확정
5. 선택 결과(보상/리스크/이벤트) 적용

## 4) 팬(Fan) 배치 규칙
- 기준점(anchor)은 화면 하단 중앙 또는 이벤트 패널 중앙
- 카드 수(`N`)에 따라 각도 간격 자동 조정
- 중앙 카드는 가장 낮은 회전값, 양끝 카드는 회전값 증가
- Hover된 카드는 확대 + 전면 이동(`z-index` 상승)

## 5) 리롤 연출 규칙
리롤 버튼 입력 시 아래 시퀀스를 실행:
1. **수렴(Collect)**: 펼쳐진 카드가 중앙으로 이동해 뭉치 형성
2. **뒤집힘(Flip)**: 카드 뭉치 전체가 뒷면으로 전환
3. **회전(Spin x3)**: 각 카드가 시계/반시계 방향으로 3회 회전
4. **복귀(Return)**: 각 카드가 자신의 팬(Fan) 위치로 이동
5. **순차 재뒤집힘(Sequential Flip)**: 카드가 한 장씩 앞면으로 돌아옴

## 6) 드래그 상호작용 규칙
- 카드는 항상 드래그 가능 상태를 유지한다.
- 드래그 중에는 포인터 위치로 카드가 즉시 추종한다.
- 포인터/클릭 해제 시 카드는 원래 팬(Fan) 기준 위치로 복귀한다.

## 7) 상태 머신(권장)
- `idle` → `fan_open` → `selecting`
- `selecting` → `reroll_collect` → `reroll_flip` → `reroll_shuffle` → `reroll_refan` → `selecting`
- 카드 선택 확정 시 `resolved`

## 8) UX 기준
- 리롤 전체 연출 시간: 0.7s ~ 1.2s 권장
- 유저 입력 락: 셔플 중 중복 입력 방지
- 카드 정보 가독성: 제목/효과/희귀도 색상 구분
- 모션 약화 옵션: 접근성 설정에서 연출 최소화 제공

## 9) 밸런스 연결
- 리롤 횟수 제한 또는 리소스(골드/마나/토큰) 소모
- 리롤 시 희귀도 가중치 변화(옵션)
- 특정 절기/주차(달력 시스템)와 연계해 카드 풀 변동 가능
