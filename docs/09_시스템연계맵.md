# 09. 시스템 연계 맵

- 버전: **ver.0.0.20(260329-시작화면HTML문서추가)**

## 1) 목적
- 시스템 간 데이터 흐름/의존 관계를 한눈에 확인하기 위한 맵 문서.

## 2) 상위 의존 구조
- 월드(01) → 정착지(02) → 국가(03)
- 월드(01) ↔ 전투(04)
- 캐릭터 프로필(05) → 렌더링(07)
- 달력(06) → 이벤트/카드(08)
- 전 시스템 → 저장/불러오기(10)
- 모드(11) ↔ 월드/전투/카드/정착지 훅

## 3) 이벤트 흐름 예시
1. 월드에서 조우 이벤트 발생 (01)
2. 전투로 전환 (04)
3. 모드 훅이 계산 단계에 개입 (11)
4. 결과가 정착지/국가 상태로 반영 (02, 03)
5. 현재 절기/주차에 따라 보정 적용 (06)
6. 다음 선택지 카드 생성 (08)
7. 턴 종료 저장 실행 (10)

## 4) 데이터 오너십
- 타일 데이터 오너: 01
- 정착지 레벨/스탯 오너: 02
- 국가/수도 규칙 오너: 03
- 전투 상태 전이 오너: 04
- 캐릭터 정체성/프로필 오너: 05
- 시간축 상태 오너: 06
- 외형 레이어/틴트 오너: 07
- 카드 선택 UX/상태머신 오너: 08
- 저장 포맷/검증 오너: 10
- 모드 훅/권한/격리 오너: 11

## 5) 변경 시 체크리스트
- 연관 문서 동시 업데이트 여부 확인
- 마스터 문서(00) 링크/요약 갱신
- 저장 포맷 영향(10) 검토
- 모드 API 영향(11) 검토
- 변경이력(99) 기록
