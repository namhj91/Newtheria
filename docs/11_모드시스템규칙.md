# 11. 모드 시스템 규칙

- 버전: **ver.0.0.20(260329-시작화면HTML문서추가)**

## 1) 목표
- 모드가 코어를 직접 수정하지 않고도 규칙/행동/계산을 확장할 수 있게 한다.
- "완전히 다른 행동과 계산"이 가능하도록 훅 기반 확장 지점을 제공한다.

## 2) 모드 구조
- `manifest.json` (id, name, version, author, dependencies)
- `scripts/` (행동/계산 확장 로직)
- `assets/` (이미지/사운드/UI)
- `data/` (밸런스/테이블 오버레이)

## 3) 실행 방식
- 모드 스크립트는 코어 런타임의 **샌드박스 VM**에서 실행
- 직접 파일 시스템/네트워크/프로세스 접근 금지
- 허용된 API를 통해서만 게임 상태 조회/변경 가능

## 4) 훅(Hook) 예시
- `onTurnStart`, `onTurnEnd`
- `onBattleInit`, `onDamageCalc`, `onSkillResolve`
- `onCardGenerate`, `onCardSelect`
- `onSettlementUpgrade`, `onNationEvent`

## 5) 권한 모델
- `read_world`, `write_world`
- `read_character`, `write_character`
- `combat_modifier`, `ui_extension`
- 모드는 명시된 권한만 요청/부여 가능

## 6) 충돌 처리
- 동일 훅 다중 모드 충돌 시 우선순위 규칙:
  1. 사용자 정렬 순서
  2. 의존성 그래프
  3. 충돌 시 로드 차단 + 로그 안내

## 7) 세이브 연동
- 세이브에 모드 ID/버전/해시 저장
- 로드 시 모드 불일치 경고 및 호환 모드(안전모드) 제공

## 8) 안전/안정성
- 실행 시간 제한(타임아웃)
- 메모리 사용량 제한
- 예외 발생 모드는 격리 비활성화
