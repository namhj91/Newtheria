# 대사 스크립트 문법 문서 (.asdf / 헤더형)

> 인코딩: **UTF-8**

현재 문법은 다음 원칙을 따른다.

- 상위 헤더 정의에는 `:` 사용
- 블록 내부 키/값에는 `=`만 사용
- 블록 종료는 `---`
- 계층은 `event -> scene -> block`

---

## 1) 헤더 문법

- `<event:이름>`
- `<scene#n:이름>` (`n`은 0 이상의 정수)
- `<block:anchor이름>` 또는 `<block>`

상위 헤더를 새로 정의하기 전까지는 현재 상위 계층이 유지된다.

---

## 2) 씬 캐스팅 + 블록 문법

`<scene#n:이름>` 바로 다음에 `{ ... }`를 쓰면 씬 기본 캐스팅을 선언할 수 있다.

```text
<scene#0:도입>
{
actor = actorId
slot_left = actorId
slot_right = actorId
focus = actorId
cast_ref = optional_bundle_id
cast_json = optional_json_string
}
---
```

scene 캐스팅 키(현재):
- `actor` : `actorId` (캐릭터 리스트/번들에서 actor 정보를 조회)
- `slot_left`, `slot_right` : 좌/우 기본 슬롯 actor id
- `focus` : 기본 포커스 actor id
- `cast_ref` : (예약) 외부 캐릭터 번들 식별자
- `cast_json` : (예약) 외부 캐릭터 번들 JSON 문자열

블록 내부는 반드시 `{` 와 `}` 로 감싼다.

```text
<block:anchor>
{
key = value
}
---
```

블록 지원 키(현재):

- `speaker` : 대화창 표시 이름(선택)
- `speaker_id` 또는 `actor` : 실제 발화 actor id (speaker 미지정 시 표시 이름 자동 추론)
- `line` : 대사 내용
- `wait` (`delay`, `pause`, `auto_next_ms`) : 자동 진행 대기 시간(ms)
- `type_speed` (`typewriter_speed`, `type_ms`) : 타자기 속도(문자당 ms)
- `particle` (`particles`, `fx_particle`, `vfx_particle`) : 파티클 프리셋 (`snow`, `ember`, `off`)
- `particle_count` : 파티클 개수 (기본 18)
- `particle_duration` : 파티클 기준 지속시간(ms, 기본 3200)
- `bg` 또는 `background` : 배경 경로
- `move_left` (`standing_left`) : 좌측 슬롯 actor id 이동
- `move_right` (`standing_right`) : 우측 슬롯 actor id 이동
- `focus` (`standing_focus`) : 포커스 actor id
- `hide` : 숨길 대상 (`actorId`, `left`, `right`, `all`, 콤마 구분)
- `standing_animation` : 스탠딩 애니메이션 (기본 전체 적용 + 범위 지정 문자열 지원)
  - 전체 공통: `standing_animation = fade`
  - 좌/우 분리(인라인): `standing_animation = left:float,right:shake`
  - 액터 지정(인라인): `standing_animation = goddess:float,pilgrim:shake`
  - 좌/우 분리(개별 키): `standing_animation_left = float`, `standing_animation_right = shake`
  - 액터 지정(개별 키): `standing_animation_actor = goddess:float,pilgrim:shake`
  - 지원 프리셋: `fade`, `float`, `zoom-in`, `shake`
- `set_layers` : actor 레이어 갱신 (`actorId:img1,img2`)
- `jump` : 점프할 anchor 이름
- `end` : 종료 여부(true/false)
- `sfx` : 효과음 경로(현재 파싱만)
- `choice` : 선택지 (`choice = 라벨 -> anchor`)
- `hidden_choice` : 히든 선택지 (`hidden_choice = 라벨 -> anchor`)

대사/화자 문자열에는 변수 템플릿을 사용할 수 있다.

- `{변수명}` 형식으로 사용 (`line`, `speaker` 등에 적용)
- 예: `line = 안녕, {player_name}.`

## 3) 기본 예시

```text
<event:오프닝>
<scene#0:도입>
{
cast_json = "{\"actors\":[{\"id\":\"goddess\",\"name\":\"아스테리아\",\"layers\":[\"assets/img/goddess.png\"]},{\"id\":\"pilgrim\",\"name\":\"순례자\",\"layers\":[\"assets/img/player.png\"]}]}"
actor = goddess
actor = pilgrim
slot_left = goddess
slot_right = pilgrim
focus = goddess
}
---

<block:intro_start>
{
actor = goddess
speaker = 별의 여신 {goddess_title}
focus = goddess
standing_animation = fade
type_speed = 24
particle = snow
particle_count = 20
particle_duration = 3600
line = 여정에 온 걸 환영해, {player_name}.
wait = 700
choice = 첫 번째 선택 -> 첫번째_선택지
choice = 두 번째 선택 -> 두번째_선택지
choice = 세 번째 선택 -> 세번째_선택지
}
---

<block:첫번째_선택지>
{
speaker_id = pilgrim
focus = pilgrim
line = 첫 번째 선택을 골랐군요.
jump = 선택종료
}
---

<block:선택종료>
{
speaker = 아스테리아
line = 이제 다음으로.
}
---
```

## 4) 진행 규칙

1. block 안에 `choice`가 있으면 사용자가 선택 버튼으로 진행
   - 모든 `choice` 카드를 버린 경우(`drag discard`)에 한해 `hidden_choice`가 있으면 히든 선택지가 노출된다.
2. `jump`가 있으면 해당 anchor로 이동
3. 둘 다 없으면 같은 scene의 다음 block으로 이동
4. scene 마지막 block이면 같은 event의 다음 scene으로 이동
5. event 마지막 scene이면 종료
6. `end = true`면 즉시 종료
7. `wait`가 있으면 해당 ms 이후 자동으로 다음 진행을 시도
8. `particle`이 있으면 해당 프리셋 파티클로 화면 오버레이를 갱신 (`off`로 해제)

---

## 5) 변수 패치 런타임

런타임 API `setVariables({ ... })`로 변수값을 주입/갱신할 수 있다.

- 예: `setVariables({ player_name: "테스터", goddess_title: "아스테리아" })`
- 변수 갱신 시 현재 block이 즉시 재렌더되어 텍스트 치환 결과가 반영된다.

---

## 6) 오류 방지 규칙 (엄격 검증)

다음 항목은 즉시 오류:

1. event 없이 scene 선언
2. scene 없이 block 선언
3. block 외부 key/value 사용
4. block 내부에서 `=` 형식이 아닌 줄
5. block에서 `{` 또는 `}` 누락/순서 오류
6. scene 중복 (`event + scene#n`)
7. scene에 block 없음
8. anchor 중복
9. jump target anchor 미존재
10. choice target anchor 미존재
11. hidden_choice target anchor 미존재

---

## 7) 스토리지 연동 메모

- `actor = id`는 우선순위에 따라 캐릭터 정보를 조회한다.
  1) scene의 `cast_json` / `cast_ref` 번들
  2) 활성 세이브 슬롯 캐릭터(`localStorage["newtheria.saveSlots.data"]`)
  3) 대표 캐릭터 카탈로그(`localStorage["newtheria.characters"]`, 여신/순례자 등 소수)
  4) 엔진 기본 임시 카탈로그
- 대화 진행 포인터는 `sessionStorage["newtheria.dialogue.progress"]`에 저장/복원된다.
