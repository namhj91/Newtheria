# 07. 캐릭터 레이어 렌더링 규칙

- 버전: **ver.0.0.19(260329-별자리라인연출추가)**

## 1) 목표
- 캐릭터 외형을 부위별 레이어 조합 방식으로 표현한다.
- 기본 에셋을 그레이스케일로 통일하고, 색상 팔레트 적용으로 다양성을 확보한다.

## 2) 레이어 구성(기본)
1. BaseBody (신체 기본)
2. Face (눈/입/표정)
3. Hair (앞머리/뒷머리)
4. Outfit (의상)
5. Armor/Accessory (방어구/장신구)
6. Weapon (무기)
7. FX (오라/상태이상/강화 이펙트)

## 3) 렌더링 순서 규칙
- 뒤 레이어부터 앞 레이어 순으로 합성한다.
- 같은 슬롯 내에서는 `zOrder` 값으로 정렬한다.
- 장비 교체 시 해당 슬롯 레이어만 교체하여 전체 재생성을 최소화한다.

## 4) 그레이스케일 + 틴트 정책
- 원본 텍스처는 그레이스케일로 제작한다.
- 렌더 시 팔레트/틴트 값을 적용해 색상 변형한다.
- 부위별 팔레트 채널 예시:
  - SkinTint, HairTint, ClothTint, MetalTint, EmissiveTint

## 5) 데이터 구조 예시
- `character.layers[]`
  - `slot` (body/hair/outfit/...)
  - `assetId`
  - `zOrder`
  - `tint` (r,g,b)
  - `visible`

## 6) 장점
- 동일 아트 재사용률 증가
- 스킨/염색/진영색 시스템 확장 용이
- 메모리 사용량 및 제작 비용 절감

## 7) 품질 기준
- 레이어 경계선 깨짐 방지(안티앨리어싱 기준)
- 팔레트 적용 시 명암 보존
- UI 썸네일/전투 스프라이트 간 색상 일관성 유지
