# 00. 문서 운영 가이드

- 버전: **ver.0.0.20(260329-시작화면HTML문서추가)**

## 1) 목적
- 문서의 읽기 순서와 수정 책임을 명확히 하여 기획 혼선을 줄인다.
- 신규 기능 추가 시 어느 문서를 먼저 수정해야 하는지 기준을 제공한다.

## 2) 권장 읽기 순서
1. `00_MASTER_SPEC.md` (전체 개요)
2. `01_월드규칙.md` (월드/타일)
3. `02_정착지규칙.md` (성장/내정)
4. `03_국가수도규칙.md` (거시 운영)
5. `04_전투연동규칙.md` (전투 연결)
6. `05_캐릭터프로필규칙.md` (캐릭터 데이터)
7. `07_캐릭터레이어렌더링규칙.md` (비주얼 파이프라인)
8. `06_달력규칙.md` (시간축/이벤트)
9. `08_카드선택연출규칙.md` (UI/연출)
10. `99_변경이력.md` (버전 히스토리)

## 3) 수정 우선순위 규칙
- 시스템 변경은 항상 `00_MASTER_SPEC.md`를 먼저 갱신한다.
- 세부 규칙 변경은 해당 영역 문서를 갱신한다.
- 변경 요약은 `99_변경이력.md`에 반드시 기록한다.

## 4) 충돌 해결 규칙
- 문서 간 내용 충돌 시 우선순위:
  1. `00_MASTER_SPEC.md`
  2. 영역별 세부 문서(01~11)
  3. `99_변경이력.md` 설명

## 5) 네이밍/버전 규칙
- 파일명 번호는 문서 성격(코어 → 세부 → 기록)을 나타낸다.
- 버전 표기는 `ver.0.0.x(YYMMDD-업데이트주제)` 형식을 유지한다.
