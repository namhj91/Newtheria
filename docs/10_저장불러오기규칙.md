# 10. 저장/불러오기 규칙


## 1) 목표
- 게임 진행 상태를 안정적으로 저장/복구한다.
- 버전 업데이트 후에도 세이브 호환성을 최대한 유지한다.

## 2) 저장 단위
- 글로벌 설정(오디오/그래픽/접근성)
- 캠페인 세이브 슬롯
- 모드 상태/활성 목록

## 3) 세이브 데이터 범위
- 월드 상태(타일/위협/영향권)
- 정착지/국가 상태
- 캐릭터/인벤토리/진행 이벤트
- 달력 상태(절기/월/주/턴)
- 카드 선택 상태(진행 중 선택지 포함)

## 4) 포맷 제안
- `saveVersion`: 세이브 포맷 버전
- `gameVersion`: 게임 빌드 버전
- `mods[]`: 활성 모드 ID/버전/해시
- `payload`: 실제 게임 상태 데이터

## 5) 로드 검증
- 세이브 무결성 체크(해시/서명)
- 필수 필드 누락 시 로드 중단
- 버전 차이 시 마이그레이션 단계 적용

## 6) 자동 저장 정책
- 턴 종료 시 자동저장(옵션)
- 중요 이벤트(전투 종료/정착지 승급/수도 변경) 시 체크포인트 저장

## 7) 실패 대응
- 로드 실패 시 직전 백업 슬롯 제안
- 손상 세이브는 격리 저장 후 복구 시도 로그 기록
